Create a thread.

Thread naming is a little complicated\+: Most systems have very small limits for the string length (Haiku has 32 bytes, Linux currently has 16, Visual C++ 6.\+0 has nine!), and possibly other arbitrary rules. You\textquotesingle{}ll have to see what happens with your system\textquotesingle{}s debugger. The name should be UTF-\/8 (but using the naming limits of C identifiers is a better bet). There are no requirements for thread naming conventions, so long as the string is null-\/terminated UTF-\/8, but these guidelines are helpful in choosing a name\+:

\href{http://stackoverflow.com/questions/149932/naming-conventions-for-threads}{\texttt{ http\+://stackoverflow.\+com/questions/149932/naming-\/conventions-\/for-\/threads}}

If a system imposes requirements, SDL will try to munge the string for it (truncate, etc), but the original string contents will be available from \doxylink{_s_d_l__thread_8h_a5c1e3618f2be2e3af69d615055defa5a}{SDL\+\_\+\+Get\+Thread\+Name()}. \Hypertarget{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}\label{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}

{\footnotesize\ttfamily DECLSPEC void SDLCALL SDL\+\_\+\+Detach\+Thread (\begin{DoxyParamCaption}\item[{SDL\+\_\+\+Thread \texorpdfstring{$\ast$}{*}}]{thread }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [extern]}}

A thread may be "{}detached"{} to signify that it should not remain until another thread has called \doxylink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{SDL\+\_\+\+Wait\+Thread()} on it. Detaching a thread is useful for long-\/running threads that nothing needs to synchronize with or further manage. When a detached thread is done, it simply goes away.

There is no way to recover the return code of a detached thread. If you need this, don\textquotesingle{}t detach the thread and instead use \doxylink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{SDL\+\_\+\+Wait\+Thread()}.

Once a thread is detached, you should usually assume the SDL\+\_\+\+Thread isn\textquotesingle{}t safe to reference again, as it will become invalid immediately upon the detached thread\textquotesingle{}s exit, instead of remaining until someone has called \doxylink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{SDL\+\_\+\+Wait\+Thread()} to finally clean it up. As such, don\textquotesingle{}t detach the same thread more than once.

If a thread has already exited when passed to \doxylink{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}{SDL\+\_\+\+Detach\+Thread()}, it will stop waiting for a call to \doxylink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{SDL\+\_\+\+Wait\+Thread()} and clean up immediately. It is not safe to detach a thread that might be used with \doxylink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{SDL\+\_\+\+Wait\+Thread()}.

You may not call \doxylink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{SDL\+\_\+\+Wait\+Thread()} on a thread that has been detached. Use either that function or this one, but not both, or behavior is undefined.

It is safe to pass NULL to this function; it is a no-\/op. \Hypertarget{_s_d_l__thread_8h_a0bf5d93ed5cb25ea59307472c62c66db}\label{_s_d_l__thread_8h_a0bf5d93ed5cb25ea59307472c62c66db}

{\footnotesize\ttfamily DECLSPEC void SDLCALL SDL\+\_\+\+Wait\+Thread (\begin{DoxyParamCaption}\item[{SDL\+\_\+\+Thread \texorpdfstring{$\ast$}{*}}]{thread,  }\item[{int \texorpdfstring{$\ast$}{*}}]{status }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [extern]}}

Wait for a thread to finish. Threads that haven\textquotesingle{}t been detached will remain (as a "{}zombie"{}) until this function cleans them up. Not doing so is a resource leak.

Once a thread has been cleaned up through this function, the SDL\+\_\+\+Thread that references it becomes invalid and should not be referenced again. As such, only one thread may call \doxylink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{SDL\+\_\+\+Wait\+Thread()} on another.

The return code for the thread function is placed in the area pointed to by {\ttfamily status}, if {\ttfamily status} is not NULL.

You may not wait on a thread that has been used in a call to \doxylink{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}{SDL\+\_\+\+Detach\+Thread()}. Use either that function or this one, but not both, or behavior is undefined.

It is safe to pass NULL to this function; it is a no-\/op.
