\doxysubsection{Detailed Description}
Class for drawing complex polygons on the display using \doxylink{classtouchgfx_1_1_canvas_widget_renderer}{Canvas\+Widget\+Renderer}. The \doxylink{classtouchgfx_1_1_canvas_widget}{Canvas\+Widget} is used by passing it to a \doxylink{classtouchgfx_1_1_canvas}{Canvas} object, drawing the outline of the object and then having \doxylink{classtouchgfx_1_1_canvas_widget}{Canvas\+Widget} render the outline on the display using the assigned painter.

{\footnotesize\ttfamily void touchgfx\+::\+Canvas\+Widget\+::draw (\begin{DoxyParamCaption}\item[{const \mbox{\hyperlink{classtouchgfx_1_1_rect}{Rect}} \&}]{invalidated\+Area }\end{DoxyParamCaption}) const\hspace{0.3cm}{\ttfamily [virtual]}}

Draws the given invalidated area. If the underlying \doxylink{classtouchgfx_1_1_canvas_widget_renderer}{Canvas\+Widget\+Renderer} fail to render the widget (if the widget is too complex), the invalidated area is cut into smaller slices (horizontally) which are then drawn separately. If drawing a single raster line fails, that line is considered too complex and skipped (it is left blank/transparent) and drawing continues on the next raster line.

If drawing has failed at least once, which means that the number of horizontal lines draw has been reduced, the number of successfully drawn horizontal lines is remembered for the next invocation of \doxylink{classtouchgfx_1_1_canvas_widget_ab5c3dcbbd422e9d20467e9b9092b6ab9}{draw()}. A future call to \doxylink{classtouchgfx_1_1_canvas_widget_ab5c3dcbbd422e9d20467e9b9092b6ab9}{draw()} would then start off with the reduced number of horizontal lines to prevent potentially drawing the canvas widget in vain, as happened previously in \doxylink{classtouchgfx_1_1_canvas_widget_ab5c3dcbbd422e9d20467e9b9092b6ab9}{draw()}.
