1/** ***************************************************************************
2 * @file
3 * @brief Calculations for displaying the measurements
4 *
5 *
6 * Calculations for displaying the measurements
7 * ====================================================
8 *
9 * Distance Calibration
10 * -----------
11 * - Calibration involves using the given distance between the board and the wire.
12 * - Positioning of the board is crucial; it must be perpendicular to the wire during calibration.
13 * - A ruler is employed for measuring the distance between the board and the wire.
14 * - The calibration process includes taking three measurements per pad, which are executed through the GUI.
15 * - Measurements are conducted at intervals of 10 mm, 50 mm, and 100 mm.
16 * - For each pad and each set distance, the signal strength is recorded. See @ref calculate_freq_and_signalstrength for more details.
17 * - For a visual representation of the calibration setup, refer to the diagram below.
18 * @note Values are read from flash: @ref start_calibration
19 *
20 * @image html Doxygen/img/Calibration_Setup.png height=450px
21 *
22 * Current Calibration
23 * -----------
24 * - Calibration involves using the given current through the wire, the type of cable (single or three-phase), and the distance
25 * between the board and the wire.
26 * - Positioning of the board is crucial; it must be perpendicular to the wire during calibration.
27 * - A ruler is employed for measuring the distance between the board and the wire.
28 * - The calibration process includes taking one measurement per hall-sensor, which are executed through the GUI.
29 * - Measurements are conducted at different currents (1.2 A and 5 A) and different phases (single and three-phase).
30 * - For each hall-sensor, current, and phase, the signal strength is recorded. See @ref calculate_freq_and_signalstrength for more details.
31 * - For a visual representation of the calibration setup, refer to the diagram below.
32 * @note Values are read from flash: @ref start_calibration
33 *
34 * @image html Doxygen/img/Calibration_Setup_current.png height=500px
35 *
36 * Angle Calculation and Distance Approximation
37 * -------
38 * The equation for this trigonometry can be seen
39 * under the figure.
40 *
41 * @image html Doxygen/img/Angle_Calculation.png height=400px
42 *
43 * @note The arctangent function, applied to calculate the angle between pad distance readings,
44 * maps any real number to an angle within -90 to 90 degrees,
45 * reflecting the ratio of pad distance value differences to the fixed pad separation,
46 * thereby facilitating the calculation of angles up to ±90 degrees.
47 * However, visualizing these angles as traditional triangles is challenging,
48 * as extreme sensor value differences — representing steep,
49 * nearly vertical angles — do not correspond neatly to standard geometric shapes.
50 * That's why the visualization using standard geometric shapes
51 * only allows angles between -45 and 45 degrees to be visualized.
52 * Refer to the image above for a better understanding of the angle calculation.
53 *
54 * @see D_P Distance from the board center to pad center
55 *
56 * @n signal_l: Signal strength of the left pad
57 * @n signal_r: Signal strength of the right pad
58 *
59 * Coefficients Used for Distance Approximation
60 * ===================================
61 * Distance
62 * --------
63 * Approximation is done with a Laurent polynomial of 2nd degree.
64 * @n By solving the equation system with the 3 measurements, the coefficients can be calculated.
65 * Each coefficient is calculated with the following equation:
66 *
67 *
68 * @image html Doxygen/img/distance_coefficients.png height=400px
69 * @see calculate_coefficients_single_pad
70 *
71 * The coefficients are calculated for each pad. With these coefficients, the distance can be approximated.
72 * @ref calculate_distance_and_angle function calculates both distances from each pad
73 * and returns the mean value of both distances as the real distance between the board center and the wire.
74 *
75 * This approach allows the calibration and calculation to be done all on the board itself,
76 * instead of having to calculate the approximation externally using MATLAB and importing the values through a look-up table.
77 *
78 * The approximation, in comparison to the look-up table approach, looks like this:
79 * @image html Doxygen/img/approximation_plot.png height=600px
80 * Using the coefficient approach, results in a very similar approximation to the look-up table approach, while being more efficient
81 * and not requiring an external device to calculate the coefficients.
82 * @note @htmlonly
83 * <a href="../files/LookUpTable_vs_Coefficients.m">MATLAB SCRIPT</a>
84 * @endhtmlonly
85 *
86 * @n For calibration, only 3 measurements are needed.
87 * @n d1, d2, d3: Distance for each measurement
88 * @n s1, s2, s3: Signal for each measurement
89 * @n a, b, c: @ref calculate_coefficients_single_pad
90 *
91 * Current Calculation
92 * -------
93 * Approximation is done with a constant, as seen in the figure.
94 * @image html Doxygen/img/Current_Aproximation.png height=400px
95 * This approach allows only an accurate approximation for a specific current in a very limited and close range.
96 * \n The approximation is done for 1.2 A and 5 A.
97 * \n The approximation is done for each pad and for each current.
98 * \ As the phase changes the magnetic field, a different approximation is needed for each phase.
99 *
100 * @n For calibration, only 1 measurement is needed per pad, set current (1.2 A or 5 A), and phase.
101 * @n U: Signal
102 * @n r: Distance from the board center to wire
103 * @n const: @ref calculate_magnetic_coefficients
104 *
105 *
106 * @todo Optimize accuracy of approximation
107 * @todo Optimize output of accurate measurement
108 * @todo Animate shutting off screen (Currently only static image)
109 *
110 * @bug Current calculation sometimes returns NaN or inf values
111 *
112 * ----------------------------------------------------------------------------

175/**
176 * @brief Calculates the frequency and signal strength using Fast Fourier Transform (FFT).
177 *
178 * This function takes an array of input samples and performs the following steps:
179 * 1. Removes the DC component from the input samples.
180 * 2. Performs FFT on the input samples to obtain the complex output.
181 * 3. Calculates the magnitude of the complex numbers.
182 * 4. Finds the index of the maximum magnitude in the first half of the array.
183 * 5. Determines the main frequency based on the maximum magnitude index.
184 * 6. Rounds the main frequency to either 50 Hz or 60 Hz.
185 * 7. Calculates the index of the rounded frequency.
186 * 8. Calculates the signal strength at the rounded frequency.
187 * 9. Returns the calculated main frequency and signal strength in a structure.
188 *
189 * @param input_samples The array of input samples.
190 * @return The structure containing the calculated main frequency and signal strength.
191 * @see @a <a href="https://www.keil.com/pack/doc/CMSIS/DSP/html/index.html">CMSIS DSP Software Library </a>
192 */
193FFT calculate_freq_and_signalstrength(float32_t input_samples[])
194{
195 // remove DC component
196 for (uint32_t i = 0; i < SAMPLE_LEN; i++) {
197 input_samples[i] -= 2047.5;
198 }
199 // Create an array for the FFT output
200 float32_t fft_complex_output[2 * SAMPLE_LEN];
201 // Create an instance of the FFT structure
202 arm_rfft_fast_instance_f32 fft_struct;
203 // Initialize the FFT structure
204 arm_rfft_fast_init_f32(&fft_struct, SAMPLE_LEN);
205 // Perform the FFT on the input samples
206 arm_rfft_fast_f32(&fft_struct, input_samples, fft_complex_output, 0);
207 // Calculate magnitude of complex numbers
208 arm_cmplx_mag_f32(fft_complex_output, fft_complex_output, SAMPLE_LEN / 2);
209 // Find the index of the maximum magnitude in the first half of the array
210 uint32_t maxIndex;
211 float32_t maxValue;
212 arm_max_f32(fft_complex_output, SAMPLE_LEN / 2, &maxValue, &maxIndex);
213 // Create an instance of the FFT structure
214 FFT fft;
215 // Calculate the main frequency
216 float main_frequency =
217 (float)maxIndex * ((float)MEAS_get_samp_freq()) / SAMPLE_LEN;
218 uint8_t given_frequency = 0;
219 // Check if the main frequency is in the range of 45-55 Hz
220 if (main_frequency <= 55 && main_frequency >= 45)
221 {
222 // If yes, round the given frequency to 50 Hz
223 given_frequency = 50;
224 }
225 else if (main_frequency <= 65 && main_frequency >= 55)
226 {
227 // If yes, round the given frequency to 60 Hz
228 given_frequency = 60;
229 }
230 else
231 given_frequency = 0;
232 // calculate the index of the given frequency
233 uint32_t index = (uint32_t)given_frequency * SAMPLE_LEN / MEAS_get_samp_freq();
234 // set the signal strength to 0
235 float signal_strength = 0;
236 // check if the given frequency is 50 or 60 Hz
237 if (given_frequency != 0)
238 {
239 // calculate the signal strength
240 signal_strength = 2 * fft_complex_output[index] / SAMPLE_LEN;
241 fft.error = CALC_ERROR_NONE;
242 }
243 else
244 fft.error = CALC_ERROR_FREQUENCY; // if the given frequency is not 50 or 60 Hz
245 // return the calculated values
246 fft.main_freq = main_frequency;
247 fft.signal_strength = signal_strength;
248 return fft;
249}

251/**
252 * @brief Performs accurate FFT calculation for signal strength and main frequency of each channel.
253 *
254 * This function calculates the signal strength and main frequency for each channel using FFT.
255 * It performs accurate measurement loops and calculates the mean and standard deviation of the signal strength.
256 * If the standard deviation is larger than 100, it sets the error flag accordingly.
257 *
258 * @return The calculated ACCU_FFT structure containing the signal strength, standard deviation, and error flag.
259 */
260ACCU_FFT accurate_FFT(void){
261 // Create an instance of the ACCU_FFT structure
262 ACCU_FFT accu_fft;
263 // Create an instance of the FFT structure
264 FFT fft;
265
266 static float32_t accu_strength_PR[CALIBRATION_MEASUREMENT_LOOPS];
267 static float32_t accu_strength_PL[CALIBRATION_MEASUREMENT_LOOPS];
268 static float32_t accu_strength_HSR[CALIBRATION_MEASUREMENT_LOOPS];
269 static float32_t accu_strength_HSL[CALIBRATION_MEASUREMENT_LOOPS];
270
271 uint32_t* samples;
272
273 static float32_t samples_PR[SAMPLE_LEN];
274 static float32_t samples_PL[SAMPLE_LEN];
275 static float32_t samples_HSR[SAMPLE_LEN];
276 static float32_t samples_HSL[SAMPLE_LEN];
277
278 for (uint8_t i = 0; i < CALIBRATION_MEASUREMENT_LOOPS; i++)
279 {
280 // Start the measurement
281 samples = MEAS_start_measure();
282
283 // Convert the samples to the specific channel
284 for (uint32_t j = 0; j < SAMPLE_LEN; j++)
285 {
286 samples_PR[j] = (float32_t)samples[j * CHANNEL_NUM];
287 samples_PL[j] = (float32_t)samples[j * CHANNEL_NUM + 1];
288 samples_HSR[j] = (float32_t)samples[j * CHANNEL_NUM + 2];
289 samples_HSL[j] = (float32_t)samples[j * CHANNEL_NUM + 3];
290 }
291 // Calculate the Signal Strength and the main frequency for each channel using FFT
292
293 fft = calculate_freq_and_signalstrength(samples_PR);
294 accu_strength_PR[i] = fft.signal_strength;
295 fft = calculate_freq_and_signalstrength(samples_PL);
296 accu_strength_PL[i] = fft.signal_strength;
297 fft = calculate_freq_and_signalstrength(samples_HSR);
298 accu_strength_HSR[i] = fft.signal_strength;
299 fft = calculate_freq_and_signalstrength(samples_HSL);
300 accu_strength_HSL[i] = fft.signal_strength;
301 //current[i] = single_meas.current;
302 }
303 arm_mean_f32(accu_strength_PR, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_pr);
304 arm_std_f32(accu_strength_PR, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_pr_std_dev);
305 arm_mean_f32(accu_strength_PL, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_pl);
306 arm_std_f32(accu_strength_PL, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_pl_std_dev);
307 arm_mean_f32(accu_strength_HSR, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_hsr);
308 arm_std_f32(accu_strength_HSR, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_hsr_std_dev);
309 arm_mean_f32(accu_strength_HSL, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_hsl);
310 arm_std_f32(accu_strength_HSL, CALIBRATION_MEASUREMENT_LOOPS, &accu_fft.signal_strength_hsl_std_dev);
311 // Check if the standard deviation is larger than 100
312 if (accu_fft.signal_strength_pr_std_dev > 100 || accu_fft.signal_strength_pl_std_dev > 100 \
313 || accu_fft.signal_strength_hsr_std_dev > 100 || accu_fft.signal_strength_hsl_std_dev > 100)
314 accu_fft.error = CALC_ERROR_DEVIATION_TOO_HIGH;
315 else
316 accu_fft.error = CALC_ERROR_NONE;
317 // Return the calculated values
318 return accu_fft;
319}

321/**
322 * @brief Perform a single measurement for a specific phase.
323 *
324 * This function starts the measurement, converts the samples to the specific channel,
325 * calculates the signal strength and main frequency for each channel using FFT,
326 * calculates the distance and angle, and calculates the current using the magnetic coefficients.
327 * It also performs error testing and optimization on the calculated values.
328 *
329 * @param Phase The phase for which the measurement is performed (SINGLE_PHASE or THREE_PHASE).
330 * @return The calculated values including frequency, distance, angle, current, and error status.
331 */
332SINGLE_MEAS single_measurement(uint8_t Phase)
333{
334 // Start the measurement
335 uint32_t* samples = MEAS_start_measure();
336 // Create an instance of the SINGLE_MEAS structure
337 SINGLE_MEAS single_meas;
338 // Create an instance of the FFT structure
339 FFT fft;
340
341 // Set the error to none
342 single_meas.error = CALC_ERROR_NONE;
343
344 static float32_t samples_PR[SAMPLE_LEN];
345 static float32_t samples_PL[SAMPLE_LEN];
346 static float32_t samples_HSR[SAMPLE_LEN];
347 static float32_t samples_HSL[SAMPLE_LEN];
348
349 float32_t current_l = 0;
350 float32_t current_r = 0;
351
352 // Convert the samples to the specific channel
353 for (uint32_t i = 0; i < 64; i++)
354 {
355 samples_PR[i] = (float32_t)samples[(i * CHANNEL_NUM)];
356 samples_PL[i] = (float32_t)samples[(i * CHANNEL_NUM) + 1];
357 samples_HSR[i] = (float32_t)samples[(i * CHANNEL_NUM) + 2];
358 samples_HSL[i] = (float32_t)samples[(i * CHANNEL_NUM) + 3];
359 }
360 // Calculate the Signal Strength and the main frequency for each channel using FFT
361 fft = calculate_freq_and_signalstrength(samples_PR);
362 float32_t signal_strength_PR = fft.signal_strength;
363 float32_t main_frequency_PR = fft.main_freq;
364 fft = calculate_freq_and_signalstrength(samples_PL);
365 float32_t signal_strength_PL = fft.signal_strength;
366 float32_t main_frequency_PL = fft.main_freq;
367 fft = calculate_freq_and_signalstrength(samples_HSR);
368 float32_t signal_strength_HSR = fft.signal_strength;
369 fft = calculate_freq_and_signalstrength(samples_HSL);
370 float32_t signal_strength_HSL = fft.signal_strength;
371 // Calculate the main frequency
372 single_meas.frequency = (main_frequency_PR + main_frequency_PL) / 2;
373 // Calculate distance and angle
374 DISTANCE_ANGLE dist_angle = calculate_distance_and_angle(signal_strength_PR, signal_strength_PL);
375 single_meas.distance = dist_angle.distance;
376 single_meas.angle = dist_angle.angle;
377
378 // Calculate the current using the magnetic coefficients for 5A
379 if(Phase == SINGLE_PHASE)
380 {
381 current_l = a_magn2_l_1P * single_meas.distance * signal_strength_HSL;
382 current_r = a_magn2_r_1P * single_meas.distance * signal_strength_HSR;
383 single_meas.current = (current_l + current_r) / 2;
384 }
385 else if(Phase == THREE_PHASE)
386 {
387 if (signal_strength_HSL > signal_strength_HSR)
388 single_meas.current = a_magn2_l_3P * single_meas.distance * signal_strength_HSL;
389 else
390 single_meas.current = a_magn2_r_3P * single_meas.distance * signal_strength_HSR;
391 }
392
393 // Check if the current is smaller than 4 A -> use the magnetic coefficients for 1.2 A
394 if (single_meas.current < 4)
395 {
396 if(Phase == SINGLE_PHASE)
397 {
398 current_l = a_magn1_l_1P * single_meas.distance * signal_strength_HSL;
399 current_r = a_magn1_r_1P * single_meas.distance * signal_strength_HSR;
400 single_meas.current = (current_l + current_r) / 2;
401 }
402 else if(Phase == THREE_PHASE)
403 {
404 if (signal_strength_HSL > signal_strength_HSR)
405 single_meas.current = a_magn1_l_3P * single_meas.distance * signal_strength_HSL;
406 else
407 single_meas.current = a_magn1_r_3P * single_meas.distance * signal_strength_HSR;
408 }
409 }
410
411 // ERROR TESTING:
412
413 // Check if distance is close enough for current calculation and if the current is out of the range of 0-10 A
414 if (single_meas.distance > 20 || (single_meas.current <= 0 && single_meas.current >= 10))
415 single_meas.error = CALC_ERROR_TOO_FAR_AWAY;
416
417 // Check if the current is larger than 10 A -> CALL ERROR OVERCURRENT
418 if (single_meas.current > 10)
419 single_meas.error = CALC_ERROR_OVERCURRENT;
420
421 // Check if the distance is larger than 300 mm or smaller than 0 -> CALL ERROR DISCONNECT
422 if (single_meas.distance > 250)
423 {
424 single_meas.error = CALC_ERROR_DISCONNECT;
425 single_meas.angle = 0;
426 }
427
428 // Check if the frequency is 0 -> CALL ERROR FREQUENCY
429 if (single_meas.frequency == 0)
430 single_meas.error = CALC_ERROR_FREQUENCY;
431
432 // OPTIMIZE:
433 // Limit the angle to +/- 90 degrees
434 if (single_meas.angle > 90)
435 single_meas.angle = 90;
436 else if (single_meas.angle < -90)
437 single_meas.angle = -90;
438
439 // BUZZER LED
440 if(single_meas.distance > 0 && single_meas.distance < 200 && (State_BUZZER == false))
441 TIM14->CCR1 = 50 + (single_meas.distance/4);
442 else
443 TIM14->CCR1 = 0;
444
445 if(single_meas.distance > 0 && single_meas.distance < 200 && (State_LED == false))
446 TIM5->CCR1 = abs(100 - (int)(single_meas.distance/2));
447 else
448 TIM5->CCR1 = 0;
449 // Return the calculated values
450 return single_meas;
451}

453/**
454 * @brief Performs accurate measurement of various parameters.
455 *
456 * This function calculates the mean value and standard deviation of distance, angle, frequency, and current
457 * based on multiple single measurements. It checks for errors in the single measurements and sets the
458 * appropriate error flags. If the standard deviation of any parameter exceeds 100, it sets the
459 * error_accu flag to CALC_ERROR_DEVIATION_TOO_HIGH.
460 *
461 * @param Phase The phase for which the accurate measurement is performed.
462 * @return ACCU_MEAS The structure containing the calculated values and error flags.
463 */
464ACCU_MEAS accurate_measurement(uint8_t Phase)
465{
466 // Create an instance of the ACCU_MEAS structure
467 ACCU_MEAS accu_meas;
468
469 // set the error to none
470 accu_meas.error_accu = CALC_ERROR_NONE;
471
472 static float32_t distance[ACCURATE_MEASUREMENT_LOOPS];
473 static float32_t angle[ACCURATE_MEASUREMENT_LOOPS];
474 static float32_t frequency[ACCURATE_MEASUREMENT_LOOPS];
475 static float32_t current[ACCURATE_MEASUREMENT_LOOPS];
476
477 for(uint8_t i = 0; i < ACCURATE_MEASUREMENT_LOOPS; i++){
478 SINGLE_MEAS single_meas = single_measurement(Phase);
479 distance[i] = single_meas.distance;
480 angle[i] = single_meas.angle;
481 frequency[i] = single_meas.frequency;
482 current[i] = single_meas.current;
483
484 // Check if there is an error in the single measurement
485 if (single_meas.error != CALC_ERROR_NONE){
486 accu_meas.error_single = single_meas.error;
487 }
488 }
489 // Calculate the mean value of the distance
490 arm_mean_f32(distance, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.distance);
491 // Calculate standard deviation of the distance
492 arm_std_f32(distance, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.distance_std_dev);
493 // Calculate the mean value of the angle
494 arm_mean_f32(angle, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.angle);
495 // Calculate standard deviation of the angle
496 arm_std_f32(angle, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.angle_std_dev);
497 // Calculate the mean value of the frequency
498 arm_mean_f32(frequency, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.frequency);
499 // Calculate standard deviation of the frequency
500 arm_std_f32(frequency, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.frequency_std_dev);
501 // Calculate the mean value of the current
502 arm_mean_f32(current, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.current);
503 // Calculate standard deviation of the current
504 arm_std_f32(current, ACCURATE_MEASUREMENT_LOOPS, &accu_meas.current_std_dev);
505
506 // Check if the standard deviation is larger than 100
507 if (accu_meas.distance_std_dev > 100 || accu_meas.angle_std_dev > 100 || \
508 accu_meas.frequency_std_dev > 100 || accu_meas.current_std_dev > 100)
509 accu_meas.error_accu = CALC_ERROR_DEVIATION_TOO_HIGH;
510 else
511 accu_meas.error_accu = CALC_ERROR_NONE;
512
513 // Return the calculated values
514 return accu_meas;
515}

517/**
518 * @brief Calculates the distance and angle based on signal strengths.
519 *
520 * This function takes the signal strengths of the right and left signals and calculates
521 * the distance and angle using predefined coefficients. The calculated values are stored
522 * in a DISTANCE_ANGLE structure and returned.
523 *
524 * @param signal_strength_r The signal strength of the right signal.
525 * @param signal_strength_l The signal strength of the left signal.
526 * @return The calculated distance and angle.
527 */
528DISTANCE_ANGLE calculate_distance_and_angle(float32_t signal_strength_r, float32_t signal_strength_l)
529{
530
531 // Create an instance of the DISTANCE_ANGLE structure
532 DISTANCE_ANGLE dist_angle;
533
534 // Calculate the distance using the coefficients and the actual signal strength
535 float32_t calc_distance = a_r + (b_r / signal_strength_r) + (c_r / POW2(signal_strength_r));
536 // Set the calculated values
537 dist_angle.distance_r = calc_distance;
538
539 // Calculate the distance using the coefficients and the actual signal strength
540 calc_distance = a_l + (b_l / signal_strength_l) + (c_l / POW2(signal_strength_l));
541 // Set the calculated values
542 dist_angle.distance_l = calc_distance;
543
544 // Calculate the distance
545 dist_angle.distance = (dist_angle.distance_l + dist_angle.distance_r) / 2;
546
547 // Calculate the angle using the trigonometry
548 dist_angle.angle = atanf((dist_angle.distance_l - dist_angle.distance_r) / (2 * D_P)) * (180 / PI);
549
550 // Return the calculated values
551 return dist_angle;
552}

596/**
597 * Calculates the coefficients for the distance approximation from an approximation based on a second degree polynomial.
598 *
599 * @param s An array of three float32_t values representing x coordinates: x_0, x_1, x_2.
600 * @param d An array of three float32_t values representing y coordinates: y_0, y_1, y_2.
601 * @param a Pointer to a float32_t variable to store the calculated coefficient 'a'.
602 * @param b Pointer to a float32_t variable to store the calculated coefficient 'b'.
603 * @param c Pointer to a float32_t variable to store the calculated coefficient 'c'.
604 */
605void calculate_coefficients_single_pad(float32_t s[], float32_t d[], float32_t* a, float32_t* b, float32_t* c)
606{
607 // s = x_0, x_1, x_2 d= y_0, y_1, y_2
608 // Calculate the coefficients for the distance approximation from an approximation based on a second degree polynomial
609 // a=((x_0^(2)*(x_1-x_2)*y_0-x_0*(x_1^(2)*y_1-x_2^(2)*y_2)+x_1*(x_1*y_1-x_2*y_2)*x_2)/((x_0^(2)-x_0*(x_1+x_2)+x_1*x_2)*(x_1-x_2)))
610 // -> Result of Nspire CAS
611
612 *a = (POW2(s[0]) * (s[1] - s[2]) * d[0] - s[0] * (POW2(s[1]) * d[1] - POW2(s[2]) * d[2]) + s[1] * (s[1] * d[1] - s[2] * d[2]) * s[2])
613 / ((POW2(s[0]) - s[0] * (s[1] + s[2]) + s[1] * s[2]) * (s[1] - s[2]));
614 // b = ((−(x_0^(2)*(x_1^(2)*(y_0-y_1)-x_2^(2)*(y_0-y_2))+x_1^(2)*x_2^(2)*(y_1-y_2)))/((x_0^(2)-x_0*(x_1+x_2)+x_1*x_2)*(x_1-x_2)))
615 // -> Result of Nspire CAS
616
617 *b = (-(POW2(s[0]) * (POW2(s[1]) * (d[0] - d[1]) - POW2(s[2]) * (d[0] - d[2])) + POW2(s[1]) * POW2(s[2]) * (d[1] - d[2])))
618 / ((POW2(s[0]) - s[0] * (s[1] + s[2]) + s[1] * s[2]) * (s[1] - s[2]));
619
620 // c=((x_0*(x_0*(x_1*(y_0-y_1)-x_2*(y_0-y_2))+x_1*x_2*(y_1-y_2))*x_1*x_2)/((x_0^(2)-x_0*(x_1+x_2)+x_1*x_2)*(x_1-x_2)))
621 // -> Result of Nspire CAS
622 *c = ((s[0] * (s[0] * (s[1] * (d[0] - d[1]) - s[2] * (d[0] - d[2])) + s[1] * s[2] * (d[1] - d[2])) * s[1] * s[2]))
623 / ((POW2(s[0]) - s[0] * (s[1] + s[2]) + s[1] * s[2]) * (s[1] - s[2]));
624}
