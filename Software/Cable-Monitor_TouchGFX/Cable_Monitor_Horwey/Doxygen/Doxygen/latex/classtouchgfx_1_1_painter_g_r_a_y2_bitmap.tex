\doxysubsection{Detailed Description}
\doxylink{classtouchgfx_1_1_painter_g_r_a_y2_bitmap}{Painter\+GRAY2\+Bitmap} will take the color for a given point in the shape from a bitmap. Please be aware, the the bitmap is used by the \doxylink{classtouchgfx_1_1_canvas_widget_renderer}{Canvas\+Widget\+Renderer} (not \doxylink{classtouchgfx_1_1_shape}{Shape}), so any rotation you might specify for a \doxylink{classtouchgfx_1_1_canvas}{Canvas} \doxylink{classtouchgfx_1_1_widget}{Widget} (e.\+g. \doxylink{classtouchgfx_1_1_shape}{Shape}) is not applied to the bitmap as CWR is not aware of this rotation.

Paints a streak of pixels (all with the same alpha) in the framebuffer. The first pixel to paint is at \textquotesingle{}offset\textquotesingle{} relative to \textquotesingle{}destination\textquotesingle{} (to be able to support 1bpp, 2bpp and 4bpp). The first pixel to paint is at the given \textquotesingle{}widgetX\textquotesingle{}, \textquotesingle{}widgetY\textquotesingle{} coordinate. The number of pixels to paint is \textquotesingle{}count\textquotesingle{} and the alpha to apply is \textquotesingle{}alpha\textquotesingle{}.

Note that the widgetX, widgetY is relative to the widget in the framebuffer, not the display. To convert the coordinate to display widget coordinates, use \doxylink{classtouchgfx_1_1_abstract_painter_a342e6a1866ecc21da71b23cc94c3d55b}{framebuffer\+To\+Display()}.
