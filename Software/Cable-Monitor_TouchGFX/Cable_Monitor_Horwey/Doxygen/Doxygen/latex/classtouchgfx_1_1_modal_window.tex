\doxysubsection{Detailed Description}
\doxylink{classtouchgfx_1_1_container}{Container} for displaying a modal window and hijacking touch event and prevent them from reaching the underlying view and widgets.

The container has a background image and a surrounding box that acts as a shade on top of the rest of the screen. The background image must be set (using \doxylink{classtouchgfx_1_1_modal_window_ab69031dd87eb39101f435c4a2a4cb429}{set\+Background()}) and the shade can be adjusted (using \doxylink{classtouchgfx_1_1_modal_window_a4788d1277016d14eb692e50e3a4d169d}{set\+Shade\+Alpha()} and \doxylink{classtouchgfx_1_1_modal_window_af35ae84d28e8513daf06daf8713dafd8}{set\+Shade\+Color()}).

The \doxylink{classtouchgfx_1_1_modal_window}{Modal\+Window} can either be used directly by adding widgets/containers to the \doxylink{classtouchgfx_1_1_modal_window}{Modal\+Window} from your view or by sub-\/classing it if you need a specific \doxylink{classtouchgfx_1_1_modal_window}{Modal\+Window} with predefined behavior across your application.

The \doxylink{classtouchgfx_1_1_modal_window}{Modal\+Window} should be instantiated in the view class and added as the last element (to always be on top, i.\+e. be modal). The \doxylink{classtouchgfx_1_1_modal_window}{Modal\+Window} will fill up the entire screen so it should always be placed at x=0, y=0 on the display.

To control the visibility of the \doxylink{classtouchgfx_1_1_modal_window}{Modal\+Window} use the show and hide methods.
