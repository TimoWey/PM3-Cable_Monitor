\doxysubsection{Detailed Description}
\subsubsection*{template$<$class T1 = void, class T2 = void, class T3 = void$>$\newline
class touchgfx\+::\+Generic\+Callback$<$ T1, T2, T3 $>$}
\doxylink{classtouchgfx_1_1_generic_callback}{Generic\+Callback} is the base class for callbacks.

The reason this base class exists, is that a normal \doxylink{structtouchgfx_1_1_callback}{Callback} requires the class type where the callback function resides to be known. This is problematic for ie. framework widgets like \doxylink{classtouchgfx_1_1_abstract_button}{Abstract\+Button}, on which it should be possible to register a callback on object types that are user-\/specific and thus unknown to \doxylink{classtouchgfx_1_1_abstract_button}{Abstract\+Button}. This is solved by having \doxylink{classtouchgfx_1_1_abstract_button}{Abstract\+Button} contain a pointer to a \doxylink{classtouchgfx_1_1_generic_callback}{Generic\+Callback} instead. This pointer must then be initialized to point on an instance of \doxylink{structtouchgfx_1_1_callback}{Callback}, created by the user, which is initialized with the appropriate object type.
