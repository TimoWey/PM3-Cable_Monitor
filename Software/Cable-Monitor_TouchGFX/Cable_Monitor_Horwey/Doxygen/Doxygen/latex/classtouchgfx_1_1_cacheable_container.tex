\doxysubsection{Detailed Description}
A \doxylink{classtouchgfx_1_1_cacheable_container}{Cacheable\+Container} can be seen as a regular \doxylink{classtouchgfx_1_1_container}{Container}, i.\+e. a \doxylink{classtouchgfx_1_1_drawable}{Drawable} that can have child nodes. The z-\/order of children is determined by the order in which Drawables are added to the container -\/ the \doxylink{classtouchgfx_1_1_drawable}{Drawable} added last will be front-\/most on the screen.

The important difference is that a \doxylink{classtouchgfx_1_1_cacheable_container}{Cacheable\+Container} can also render its content to a dynamic bitmap which can then be used as a texture in subsequent drawing operations, either as a simple \doxylink{classtouchgfx_1_1_image}{Image} or in a \doxylink{classtouchgfx_1_1_texture_mapper}{Texture\+Mapper}. If the bitmap format of the dynamic bitmap differs from the format of the current \doxylink{classtouchgfx_1_1_l_c_d}{LCD}, the \doxylink{classtouchgfx_1_1_l_c_d}{LCD} from drawing the bitmap must be setup using \doxylink{classtouchgfx_1_1_h_a_l_ae329b58a0d1e461578ea8abf84d5f459}{HAL\+::set\+Auxiliary\+LCD()}.
